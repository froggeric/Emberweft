## A

### Affine Transform

A linear transformation (rotation, scale, translation) applied to coordinates before passing them to a variation function. Affine transforms are represented as a 2×3 matrix (for 2D) or a 3×4 matrix (for 3D). Each transform in a flame genome has its own affine coefficients, enabling complex spatial mappings. See [`Transform`](#transform) and [`Variation`](#variation).

### Audio-Reactive Rendering

A rendering mode where genome parameters are modulated in real-time based on audio features (beat, tempo, frequency spectrum). Used for music-video generation and live VJ performances. Audio analysis extracts onsets and BPM, which are mapped to parameters like zoom, color speed, and variation weights. See [`Beat`](#beat), [`BPM`](#bpm), [`Onset`](#onset).

---

## B

### Beat

A rhythmic pulse in audio detected through onset analysis. Beats are used in audio-reactive rendering to trigger parameter changes or visual effects. Beat detection typically uses a combination of energy thresholding and pattern matching in the frequency domain. See [`BPM`](#bpm), [`Onset`](#onset).

### BPM

Beats Per Minute — the tempo of an audio track, measured in beats per minute. BPM is estimated from the interval between detected onsets and used to synchronize visual transitions with musical rhythm. Typical values range from 60 (slow) to 180 (fast). See [`Beat`](#beat), [`Onset`](#onset).

---

## C

### Center

The focal point of the fractal flame, specified in world coordinates (usually [-1, 1] range). The camera looks at this point, and zoom is applied relative to it. Changing the center enables panning around the flame. See [`Zoom`](#zoom), [`Scale`](#pixel-perunit--scale).

### Chaos Game

The core algorithm for rendering fractal flames. A point is iteratively transformed by randomly selecting transforms (weighted by their probability) and applying their affine and variation functions. After millions of iterations, the points converge to the attractor (the flame shape). Points are histogrammed based on their final color and position. See [`Iterated Function System (IFS)`](#iterated-function-system-ifs), [`Histogram`](#histogram), [`Iterations`](#iterations).

### Density Estimation

A technique to improve image quality by estimating the local density of points in the histogram. Log-density estimation divides each histogram bin's count by the total points, reducing banding and artifacts. This is the standard approach in fractal flame rendering. See [`Histogram`](#histogram), [`Log-density`](#log-density).

## F

### Final Transform

A special transform applied after all regular transforms. It typically uses only variations (no affine component) and shapes the overall color and structure of the flame. The final transform is optional but commonly used to add global effects like symmetry or smoothing. See [`Transform`](#transform), [`Variation`](#variation).

### Flame

A fractal image generated by the chaos game algorithm. Flames are defined by a genome (transforms, variations, palette) and rendered through iteration and density estimation. The term comes from the visual appearance of early flames, which resembled fire. See [`Genome`](#genome), [`Chaos Game`](#chaos-game).

### Flock

A collection of sheep (flame genomes) that can be played in sequence. In the original Electric Sheep, the flock was a shared, server-maintained set. In Emberweft, a flock refers to any collection of genomes, such as the seed library or a user-created playlist. See [`Sheep`](#sheep), [`Seed Library`](../library/seed-library.md).

### .flam3

The file format for storing flame genomes. A .flam3 file is XML text containing transforms, variations, palette, quality settings, and metadata. The format is human-readable and can be edited manually. See [`Genome`](#genome), [`Genome Format`](../rendering/genome-format.md).

---

## G

### Gamma / Tone-map

The final color processing step to convert rendered colors to displayable values. Gamma correction compensates for display non-linearity (typically gamma 2.2). Tone-mapping clamps and scales colors to the [0, 1] range, preventing clipping while preserving detail. See [`Structural Color`](#structural-color), [`HDR Pipeline`](#hdr-pipeline).

### Genome

The complete definition of a flame, including all transforms, variations, palette, and quality settings. A genome is analogous to DNA for a sheep. Genomes are serialized to .flam3 files and can be interpolated, mutated, and bred. See [`.flam3`](#flam3), [`Genome Format`](../rendering/genome-format.md).

---

## H

### HDR Pipeline

A rendering pipeline that preserves high dynamic range (HDR) values throughout rendering, outputting to HDR displays or HDR video formats. HDR requires 10-bit or higher color depth and tone-mapping that preserves brightness range. See [`Gamma / Tone-map`](#gamma--tone-map), [`Realtime Mode`](#realtime-mode).

### Histogram

A 2D grid that accumulates points from the chaos game. Each bin stores a color (sum of point colors) and a count (number of points). After iteration, the histogram is density-estimated and tone-mapped to produce the final image. Larger histograms provide higher spatial detail but consume more memory. See [`Chaos Game`](#chaos-game), [`Density Estimation`](#density-estimation).

---

## I

### Iterated Function System (IFS)

A mathematical system defined by a set of contraction mappings. Repeatedly applying these mappings (iteration) produces a fractal attractor. Fractal flames are a type of IFS where mappings include non-linear variations. See [`Chaos Game`](#chaos-game), [`Transform`](#transform).

### Iterations

The number of points drawn in the chaos game per frame (or per pixel). More iterations produce smoother, more detailed flames but require more computation. Quality is primarily controlled by adjusting iterations. Typical values range from 500 (low quality) to 5000 (high quality). See [`Chaos Game`](#chaos-game), [`Quality (Samples)`](#quality-samples).

---

## L

### Log-density

A technique to normalize histogram counts by taking the logarithm of the count (plus a small offset). This compresses the dynamic range, reducing banding and improving visual quality. Log-density estimation is standard in fractal flame rendering. See [`Density Estimation`](#density-estimation), [`Histogram`](#histogram).

---

## O

### Onset

A moment in time when a beat or musical event occurs. Onsets are detected by analyzing audio for sudden increases in energy or spectral change. Onsets are used to trigger visual effects in audio-reactive rendering. See [`Beat`](#beat), [`BPM`](#bpm).

---

## P

### Palette

A color gradient used to map iteration values to RGB colors. Palettes are typically defined as 256–512 RGB entries and interpolated smoothly. Colors are assigned based on the transform's color index and the iteration count. See [`Structural Color`](#structural-color).

### Pixel-per-unit / Scale

The ratio of screen pixels to world coordinates. A higher scale means more pixels per world unit (zoomed in). Scale is derived from the zoom and resolution. See [`Zoom`](#zoom), [`Center`](#center).

---

## Q

### Quality (Samples)

The rendering quality, primarily determined by the number of chaos-game iterations per pixel. Higher quality (more samples) produces smoother gradients and finer details. Quality tiers (Low, Medium, High, Ultra) correspond to iteration counts (e.g., 500, 1000, 2000, 5000). See [`Iterations`](#iterations).

---

## R

### Realtime Mode

Playback where frames are rendered on-the-fly at interactive framerates (typically 30–60 FPS). Realtime mode uses adaptive quality to maintain performance. Contrast with [`Cached/pre-rendered Mode`](#cachedpre-rendered-mode). See [`Adaptive Controller`](../playback/playback-modes.md).

---

## S

### .saver

The file extension for macOS screensaver bundles. A .saver bundle is a plugin loaded by System Preferences when the screensaver activates. Emberweft will ship as a .saver bundle for integration with macOS screensaver system. See [`ScreenSaver Bundle`](#screensaver-bundle).

### Cached/pre-rendered Mode

Playback where frames are pre-rendered and cached (or played from video files). This ensures consistent quality and framerates regardless of hardware capability. Used for export and for lower-end devices. Contrast with [`Realtime Mode`](#realtime-mode).

### ScreenSaver Bundle

A macOS plugin (`.saver` file) that runs when the system is idle. Emberweft includes a screensaver bundle that plays flame animations with reduced quality and no UI. The screensaver reads genomes from the app-group container seed library. See [`.saver`](#saver), [`Seed Library`](../library/seed-library.md).

### Sheep

A single flame genome, playable as an animation. A sheep is defined by its genome (.flam3 file) and associated metadata (title, author, rating, etc.). The term comes from the original Electric Sheep project, where each genome was a "sheep" in the "flock." See [`Genome`](#genome), [`Flock`](#flock).

### Structural Color

A coloring scheme where color is assigned based on which transform was last applied, rather than spatial position. This creates color structures that follow the fractal geometry. Each transform has a color index (0–1), and the final color is interpolated from the palette based on the index. See [`Palette`](#palette).

### Supersampling / Oversample

A technique to reduce aliasing by rendering at higher resolution and downscaling. Supersampling renders multiple samples per pixel and averages them. In fractal flames, a filter radius is applied during histogram accumulation to achieve similar effects. See [`Filter Radius`](../engineering/performance.md).

---

## T

### Transform

A function that maps a point to a new point. In fractal flames, a transform consists of an affine transformation followed by a variation function. Each transform has a weight (probability) and a color index. A flame genome typically has 2–6 transforms. See [`Affine Transform`](#affine-transform), [`Variation`](#variation), [`Final Transform`](#final-transform).

### Transition / Morph

A smooth animation from one genome to another. Transitions are achieved by interpolating parameters (affine coefficients, variation weights, palette) over time. Morphs use the same interpolation as breeding (crossover). See [`Genome Interpolation`](../rendering/transitions.md).

---

## V

### Variation

A non-linear (or sometimes linear) function applied to coordinates after the affine transform. Variations define the character of the flame. Common variations include linear (identity), spherical (folds space), sinusoidal (wavy), and swirl (spirals). Each transform can have multiple variations with different weights. See [`Transform`](#transform), [`Affine Transform`](#affine-transform).

---

## Z

### Zoom

The magnification level applied to the flame. Zoom is implemented by scaling the world coordinates relative to the center point. Higher zoom values show smaller regions of the flame in more detail. Typical zoom ranges from 0.1 (wide view) to 10 (close-up). See [`Center`](#center), [`Scale`](#pixel-perunit--scale).
